# MolarPlus Marketing Website

SEO-focused Next.js marketing site for **MolarPlus** — best dental software and dental clinic management software.

## Tech Stack

- **Next.js 14** (App Router)
- **Static export** (`output: 'export'`) for fast, crawlable HTML
- **Tailwind CSS**
- **TypeScript**

## SEO Features

- **Metadata API**: Unique `<title>`, meta description, canonical URL, Open Graph, Twitter cards per page
- **Structured data**: Organization, WebSite, SoftwareApplication, Product, ContactPage, AboutPage JSON-LD
- **Sitemap**: `/sitemap.xml` (generated by `app/sitemap.ts`)
- **Robots**: `/robots.txt` (generated by `app/robots.ts`) — allow all, no noindex in production
- **One H1 per page**, logical H2/H3
- **Internal linking**: Header nav + contextual links (e.g. homepage → features, pricing, contact)
- **Target keywords**: "Dental Clinic Management Software", "Best dental Software"

## Routes

| Route | Page |
|-------|------|
| `/` | Home (400–600 words SEO content + same UI as original) |

## Development

```bash
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## Build (static export)

```bash
npm run build
```

Output is in the `out/` directory. Deploy `out/` to any static host (e.g. Vercel).

## Deployment (Vercel)

1. Connect this directory (`website-molarplus`) to Vercel.
2. Build command: `npm run build`
3. Output directory: `out`
4. Set **NEXT_PUBLIC_APP_URL** (optional): e.g. `https://app.molarplus.com` for Login/Get Started links.

Main domain (e.g. `molarplus.com`) → this project.  
App subdomain (e.g. `app.molarplus.com`) → React frontend (dashboard).

## Assets

- Logo, mockup, mobile screens, favicons are in `public/` (copied from frontend).
- Add `public/og-image.png` (1200×630) for Open Graph/Twitter share images.
